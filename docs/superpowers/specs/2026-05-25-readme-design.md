## 4. 内容结构

### 4.1 项目简介
简要说明项目是一个前后端分离的即时通讯系统，支持用户注册登录、好友管理、实时聊天与消息加密，突出系统的综合性与工程实践价值。

### 4.2 功能概述
列出系统具备的主要能力：
- 用户注册与登录
- 用户资料维护
- 好友申请、接收与好友列表管理
- 基于 WebSocket 的实时消息通信
- 历史消息查询
- 基于本地密钥的消息加密传输与密文存储

### 4.3 技术架构
说明项目采用的主要技术：
- 前端：Vue 3、Vite、Pinia、Vue Router
- 后端：Go、Gin、Gorm
- 通信方式：HTTP + WebSocket
- 数据存储：MySQL（项目依赖 MySQL DSN 启动）

### 4.4 消息加密设计（重点章节）
本章节作为 README 的核心亮点，采用答辩说明风格，重点覆盖以下内容：
- 前端浏览器基于 Web Crypto API 生成 RSA 密钥对
- 私钥仅保存在用户本地浏览器存储中，不上传服务端
- 公钥通过接口上传，用于他人向该用户发送加密消息
- 发送消息时，前端分别使用发送者公钥和接收者公钥各加密一份消息内容
- 服务端负责存储和转发密文，不直接处理消息明文
- 接收方与发送方都可以使用各自私钥解密对应密文，便于聊天展示与历史消息查看
- 明确说明本项目具备端到端加密的基本特征，但尚未达到工业级完整 E2EE 体系

### 4.5 项目亮点与特色
除消息加密外，还应总结：
- 前后端分离架构清晰
- 基于 JWT 的身份认证机制
- WebSocket 支持实时消息推送
- 好友关系校验后才允许发起聊天
- 兼顾功能完整性与安全性实践

### 4.7 快速启动
提供最少但完整的运行说明：

- 启动后端服务
- 安装并启动前端服务
- 说明前端 WebSocket 地址可通过环境变量覆盖
